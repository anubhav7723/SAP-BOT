1. Enterprise Structure Tables
Table Short Name Description & Usage
VKORG Sales Organization
Defines top-level selling unit. Links to Company Code (BUKRS). Controls pricing, document types.
TVKOV
Distribution Channel
How products reach customers: Direct, Wholesale, Retail. Assigned to Sales Org.
TVTA
Sales Area
Combination of Sales Org + Distribution Channel + Division. Core SD entity.
T001W
Plant
Manufacturing/storage location. Linked to Company Code. Used in availability check.
TVST
Shipping Point
Physical dispatch point. Determined automatically based on loading group, shipping condition, plant.
T001
Company Code
FI organizational unit. All SD transactions ultimately post to a Company Code.
TVSB
Shipping Condition
Controls automatic shipping point determination. Maintained in customer master.
TVGR
Transportation Group Groups materials for route determination. Maintained in material master.
T024E
Purchasing Org
Used in inter-company and third-party sales scenarios for purchase orders.
2. Customer Master Tables (KNA1 / Business Partner)
Table Short Name Description & Usage
KNA1
General Data
Name, address, language, country, tax number. Created once per client.
KNB1
Company Code Data
Reconciliation account, payment terms, dunning area. One record per company code.
KNVV
Sales Area Data
Sales office, group, shipping condition, pricing procedure, customer stats group.
KNVP
Partner Functions
SP (Sold-to), SH (Ship-to), BP (Bill-to), PY (Payer). Controls document partners.
KNVS
Shipping Data
Shipping conditions, delivery priority, max partial deliveries, order combination.
KNVD
Customer Material Info Customer-specific material numbers, delivery plant, loading group, units of measure.
KNVK
Contact Persons
Contact details per customer: name, department, phone, email.
KNEX
Legal Control
Export/import license data for foreign trade compliance.
BUT000 Business Partner
S/4HANA: Central BP master record. Replaces KNA1/LFA1. Role-based data management.
BUT100 BP Roles
Business Partner roles: FLCU00 (customer), FLVN00 (vendor). Role assignment table.

3. Material Master Tables – SD Relevant Views
Table Short Name Description & Usage
MARA
General Data
Material number, material type (MTART), division, basic UoM, weight, dimensions.
MAKT
Material Description
Short text per language. Used in order line items, invoices, output forms.
MVKE
Sales Org Data 1&2
Item category group (MTPOS), sales unit, minimum order qty, delivery plant, tax data.
MARC
Plant Data / MRP
MRP type, lot size, procurement type, availability check rule (MTVFP), lead times.
MARD
Storage Location Stock Unrestricted, quality inspection, blocked stock per storage location.
MLAN
Tax Classification
Tax classification per country/tax category. Used in pricing condition MWST.
MVKGR Material Group
Used for pricing access sequences and statistics grouping.
MARM
Units of Measure
All alternative UoMs with conversion factors (e.g., EA, KG, PAL, CTN).
MBEW
Material Valuation
Standard price, moving average price. Used for COGS posting at goods issue.
MLGN
Warehouse Mgmt Data
WM storage type, bulk storage indicator. Required for WM picking in delivery.
4. Sales Document Tables (Order Header, Item, Schedule Lines)
Table Short Name Description & Usage
VBAK
Sales Order Header Document number, sales area, document type, created by, date, pricing date, customer.
VBAP
Sales Order Item
Material, quantity, UoM, plant, item category, pricing data, schedule line data.
VBEP
Schedule Lines
Confirmed quantity, delivery date, MRP relevance, movement type, schedule line cat.
VBKD
Business Data
Payment terms, incoterms, purchase order number. Per header and item level.
VBPA
Partners
All partner functions (SP, SH, BP, PY, LS, WE, RE) for header and item.
VBFA
Document Flow
Links all related documents: inquiry → quotation → order → delivery → billing. Key for tracing.
VBUK
Header Status
Overall delivery status, billing status, rejection status, credit status at header.
VBUP
Item Status
Delivery status, billing status, goods movement status at item level.
VBAK-BSTNK PO Reference
Customer's purchase order number. Used for invoice matching and SOX compliance.
VAKPA
Address Data
Ship-to address override per sales order item. One-time customer address.

Table Short Name Description & Usage
LIKP

Delivery number, delivery type (LF/NL/LR), ship-to party, shipping point, planned GI date.
LIPS
Delivery Item
Material, delivery qty, plant, storage location, batch, serial numbers, item category.
LIDS
Delivery Item Detail
Packing data, handling unit assignment, dangerous goods classification.
VTTK

Shipment number, carrier, route, transportation planning point, forwarding agent.
VTTP

VEKP

Packing material, gross weight, volume, sealing number, tracking number.
VEPO
HU Item
Contents of each handling unit. Links to delivery item LIPS.
NSDM_V_TC Transport Connection
Last-mile delivery tracking data. Used with track-and-trace solutions.
EKKO
PO Header (3rd Party) Purchase order created automatically for third-party (item cat TAS) scenarios.
EKPO

PO line item with material, qty, vendor. Linked to sales order via VBAP-AUFNR.

Pro Tips for Using SAP SD Tables
Use SE16N to directly query any table with filters — faster than SE16 for large tables.
VBFA (Document Flow) is the most powerful tracing tool — enter any SD document number to see the complete chain.
KONV stores runtime pricing conditions from documents; KONH/KOND store master condition records.
In S/4HANA, BSEG is replaced by ACDOCA (Universal Journal) — always check the migration note for your release.
Use ST05 (SQL Trace) to discover which tables are being read during a transaction — great for performance tuning.
PRCD_ELEMENTS replaces KONV in S/4HANA 1909+. For older systems, KONV is your pricing debug table.
Always combine VBAK + VBAP + VBEP for complete sales order data — never query just one of them alone.

KEY BAPIs

BAPI_MATERIAL_SAVEDATA - Create/change material master data.
BAPI_PO_CREATE1 - Create a Purchase Order.
  Key parameters: POHEADER (header: doc type, vendor, purch org, currency),
  POITEM (line items: material, plant, quantity), POSCHEDULE (delivery
  schedule lines), POACCOUNTASSIGNMENT (for non-stock items), RETURN
  (messages), EXPPURCHASEORDER (output: new PO number).
BAPI_PO_CHANGE - Change an existing Purchase Order.
BAPI_PO_GETDETAIL / BAPI_PO_GETITEMS - Read PO header/item data.
BAPI_PR_CREATE - Create a Purchase Requisition.
BAPI_REQUISITION_CREATE - Older/alternate PR creation BAPI.
BAPI_GOODSMVT_CREATE - Post a goods movement (GR/GI/transfer posting).
  Key parameters: GOODSMVT_HEADER, GOODSMVT_ITEM (table), GOODSMVT_CODE
  (movement context code), RETURN, MATERIALDOCUMENT (output).
BAPI_INCOMINGINVOICE_CREATE - Post a vendor invoice (Logistics Invoice
  Verification, equivalent of MIRO).
BAPI_INCOMINGINVOICE_PARK - Park (pre-save) a vendor invoice.
BAPI_PURCHASEORDER_GETSTATUS - Read PO status/history.

GENERIC PATTERN FOR ALL "CREATE" BAPIs
Most create/change BAPIs require an explicit follow-up call to
BAPI_TRANSACTION_COMMIT (with WAIT = 'X' if you need synchronous confirmation)
to persist the change to the database; a plain BAPI call alone does not
commit the transaction. On error, BAPI_TRANSACTION_ROLLBACK can be called.
Always check the RETURN table (structure BAPIRET2, field TYPE = 'E'/'A' for
error/abort) before assuming success.

FUNCTION MODULES VS BAPIs
A Function Module (FM) is any reusable ABAP routine (created via SE37);
it may or may not be RFC-enabled (callable from outside the system).
A BAPI is a specific, formally released, RFC-enabled Function Module that
is part of the SAP Business Object Repository and is intended as a stable,
supported external interface (BAPIs follow naming/parameter conventions and
generally use BAPIRET2 for return messages). In short: "all BAPIs are RFC
function modules, but not all RFC function modules are BAPIs" - BAPIs carry
an additional guarantee of being an officially released, backward-compatible
interface.
